# Data Model: Admin Panel

## Entity Definitions

### AdminUser
- id: string (unique identifier)
- email: string (admin email address)
- role: string (single "admin" role)
- isActive: boolean (whether admin account is active)
- lastLoginAt: timestamp (when last authenticated)
- sessionTimeout: number (8 hours in milliseconds)

### AdminDashboardStats
- id: string (dashboard view identifier)
- totalSales: {
  - today: number,
  - week: number,
  - month: number
}
- totalOrders: {
  - pending: number,
  - shipped: number,
  - delivered: number,
  - cancelled: number
}
- totalUsers: number (registered user count)
- lowStockProducts: number (products with <5 stock)
- recentOrders: Array<Order> (last 10 orders)
- updatedAt: timestamp

### Product (enhanced for admin)
- Inherits from existing Product entity
- Additional fields for admin:
  - images: Array<string> (up to 5 image URLs)
  - variants: JSON (product variants stored as JSON)
  - createdAt: timestamp (when created by admin)
  - updatedAt: timestamp (when last modified by admin)
  - createdBy: string (admin user ID who created)
  - updatedBy: string (admin user ID who last modified)

### OrderStatusUpdate
- id: string (unique update record ID)
- orderId: string (reference to order)
- statusFrom: string (previous status)
- statusTo: string (new status)
- updatedBy: string (admin user ID)
- updatedAt: timestamp
- reason: string (optional reason for status change)

### AdminAuditLog
- id: string (unique log entry ID)
- action: string (type of admin action: create, update, delete, ban, etc.)
- entity: string (type of entity: product, order, user, etc.)
- entityId: string (ID of the entity)
- performedBy: string (admin user ID)
- timestamp: timestamp
- details: JSON (additional details about the action)

### Invoice
- id: string (unique invoice ID)
- invoiceNumber: string (formatted invoice number)
- dateRange: {
  - start: date,
  - end: date
}
- generatedAt: timestamp
- generatedBy: string (admin user ID)
- totalRevenue: number (total revenue in the date range)
- orderCount: number (number of orders in the date range)
- taxAmount: number (manually entered tax amount)
- topProducts: Array<{productId: string, quantity: number, revenue: number}>
- orders: Array<{orderId: string, items: Array, total: number}>
- format: string ("daily" | "weekly" | "monthly")

### AdminUserSession
- id: string (session ID)
- userId: string (admin user ID)
- expiresAt: timestamp (8 hours after creation)
- createdAt: timestamp
- lastActivityAt: timestamp

## State Transitions

### Order Status Transitions (Enforced)
Valid transitions:
- pending → shipped
- pending → cancelled
- shipped → delivered
- shipped → cancelled
- delivered → cancelled (with special permission)

Invalid transitions (blocked):
- delivered → shipped
- cancelled → pending
- cancelled → shipped
- cancelled → delivered

### User Ban Status Transitions
- active → banned
- banned → active

## Relationships

### Dashboard Stats <- One-to-Many -> Recent Orders
AdminDashboardStats contains references to recent orders for display

### AdminUser <- One-to-Many -> AdminAuditLog
Each admin action is logged with reference to admin user

### AdminUser <- One-to-Many -> OrderStatusUpdate
Each order status update is tracked by admin user

### AdminUser <- One-to-Many -> Invoice
Each invoice is generated by an admin user

## Validation Rules

### Product Validation (Admin)
- name: min 1 character
- description: min 10 characters
- price: positive number
- stock: integer >= 0
- category: must exist in system
- images: max 5 images, each max 10MB, valid formats (JPG, PNG, WebP)

### Invoice Validation
- dateRange: start date must be before end date
- taxAmount: non-negative number
- generatedBy: must be valid admin user

### Order Status Update Validation
- statusTo: must be a valid transition from current status
- updatedBy: must be valid admin user
- updatedAt: must be current timestamp

## API Endpoints

### GET /api/admin/stats
Returns dashboard statistics for admin overview

### GET /api/admin/products
Returns paginated product list for admin interface

### POST /api/admin/products
Creates new product via admin interface

### PUT /api/admin/products/:id
Updates existing product via admin interface

### DELETE /api/admin/products/:id
Deletes product via admin interface

### GET /api/admin/orders
Returns paginated order list for admin interface

### PUT /api/admin/orders/:id/status
Updates order status via admin interface

### GET /api/admin/users
Returns user list for admin interface

### PUT /api/admin/users/:id/ban
Bans/unbans user via admin interface

### GET /api/admin/analytics
Returns analytics data for charts

### GET /api/admin/invoices
Returns invoice data by date range

### POST /api/admin/invoices/generate
Generates new invoice for specified date range
