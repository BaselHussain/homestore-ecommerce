# Quickstart Guide: Frontend & Core Features

## Prerequisites
- Node.js 18+ installed
- npm or yarn package manager
- Access to backend API from Spec 1 (running on http://localhost:5000)

## Setup Instructions

### 1. Environment Setup
```bash
# Navigate to the project root
cd E:/e-commerce claude

# Create the frontend directory
mkdir frontend
cd frontend
```

### 2. Initialize the Next.js Project
```bash
# Create new Next.js app with TypeScript
npx create-next-app@latest . --typescript --tailwind --eslint --app --src-dir --import-alias --no-root-dir

# Install additional dependencies
npm install @radix-ui/react-slot lucide-react @radix-ui/react-dialog @radix-ui/react-dropdown-menu @radix-ui/react-toast @hookform/resolvers zod zustand react-hook-form axios
```

### 3. Setup shadcn/ui
```bash
# Initialize shadcn/ui (follow prompts to set up)
npx shadcn-ui@latest init

# Add required components
npx shadcn-ui@latest add button card dialog dropdown-menu toast input select checkbox form field textarea
```

### 5. Environment Variables
Create a `.env.local` file with the following variables:
```
NEXT_PUBLIC_API_BASE_URL=http://localhost:5000/api
NEXT_PUBLIC_APP_NAME=Homestore
```

Create a `.env.example` file:
```
NEXT_PUBLIC_API_BASE_URL=your_backend_api_url
NEXT_PUBLIC_APP_NAME=Homestore
```

### 6. Implementation Steps

#### Step 1: Replicate UI from homestore-sparkle
- Copy header, footer, and hero banner components exactly from homestore-sparkle
- Match colors, fonts, spacing exactly
- Ensure responsive behavior matches

#### Step 2: Implement API client
- Create `lib/api.ts` with axios instance
- Configure base URL from environment variables
- Add request/response interceptors

#### Step 3: Implement state management
- Create zustand stores for cart and wishlist
- Implement persistence to localStorage
- Add actions for add/remove/update items

#### Step 4: Build product pages
- Create product listing page with search functionality
- Create product detail page with zoom images and related products
- Connect to backend API endpoints

#### Step 5: Implement cart functionality
- Create cart page with quantity updates and removal
- Implement cart total calculation
- Add to cart functionality across product views

#### Step 6: Implement wishlist functionality
- Create wishlist page
- Add to/remove from wishlist functionality

#### Step 7: Build checkout flow
- Create multi-step checkout form
- Implement shipping address collection
- Add simulated payment processing

### 7. Run the Application
```bash
# Install dependencies
npm install

# Run in development mode
npm run dev

# Build for production
npm run build

# Run in production mode
npm start
```

### 8. Testing the Implementation
```bash
# Visit the application
open http://localhost:3000

# Test the core flows:
# 1. Browse products → search "sofa" → verify results
# 2. View PDP → add to cart → go to cart → update quantity → proceed to checkout → simulate payment → success
# 3. Add to wishlist → view wishlist → remove
# 4. UI matches homestore-sparkle demo (visual check)
```

### 9. Key Integrations
- API endpoints from Spec 1 for all data operations
- Responsive design matching homestore-sparkle
- Form validation with react-hook-form and zod
- State management with zustand for cart/wishlist
- UI components from shadcn/ui
