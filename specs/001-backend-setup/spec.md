## User Scenarios & Testing *(mandatory)*

### User Story 1 - Browse Products (Priority: P1)

As a customer visiting the Homestore website, I want to view a list of available products so that I can find items I'm interested in purchasing.

**Why this priority**: This is fundamental to e-commerce functionality - users need to see products before they can purchase them. Without this, the entire shopping experience is impossible.

**Independent Test**: Can be fully tested by making a GET request to /api/products endpoint and receiving a list of products with name, description, price, and images. The feature delivers the core catalog browsing capability.

**Acceptance Scenarios**:

1. **Given** the backend server is running, **When** a customer requests the product catalog, **Then** they receive a complete list of available products with relevant details (name, price, image, category)
2. **Given** the backend server is running, **When** a customer requests a specific product by ID, **Then** they receive detailed information about that product

---

### User Story 2 - Manage Shopping Cart (Priority: P1)

As a customer, I want to add products to my shopping cart and view my cart items so that I can track items I plan to purchase.

**Why this priority**: This is essential e-commerce functionality that enables the purchasing process. Without cart management, users cannot accumulate items for checkout.

**Independent Test**: Can be fully tested by making POST request to /api/cart to add items and GET request to /api/cart to retrieve them. The feature delivers core shopping cart functionality.

**Acceptance Scenarios**:

1. **Given** a customer has selected a product, **When** they add it to their cart, **Then** the item is successfully stored in their cart with the correct quantity
2. **Given** a customer has items in their cart, **When** they view their cart, **Then** they see all items with names, prices, quantities, and subtotals

---

### User Story 3 - Wishlist Management (Priority: P2)

As a customer, I want to add products to my wishlist so that I can save them for future purchase without committing to buy now.

**Why this priority**: This enhances user engagement and supports the shopping experience by allowing users to save items for later, which can increase conversion rates.

**Independent Test**: Can be fully tested by making POST request to /api/wishlist to add items and GET request to /api/wishlist to retrieve them. The feature delivers the ability to save products for future reference.

**Acceptance Scenarios**:

1. **Given** a customer is browsing a product, **When** they add it to their wishlist, **Then** the item is successfully stored in their wishlist

---

### User Story 4 - Complete Purchase Process (Priority: P1)

As a customer, I want to be able to place an order so that I can complete my purchase and receive the items.

**Why this priority**: This completes the core e-commerce transaction cycle. Without order placement, the shopping experience is incomplete.

**Independent Test**: Can be fully tested by making POST request to /api/orders with cart items and shipping information. The feature delivers the ability to finalize purchases.

**Acceptance Scenarios**:

1. **Given** a customer has items in their cart, **When** they submit an order, **Then** the order is successfully recorded in the system with status and shipping details

---

### Edge Cases

- What happens when the product database is empty? The system should return an empty list rather than an error.
- How does the system handle requests for products that don't exist? The system should return a 404 error with a descriptive message.
- What happens when a user tries to add more quantity of a product than is available in stock? The system should validate stock availability before adding to cart.
- How does the system handle database connection failures? The system should return appropriate error responses when unable to connect to the database.
- What happens when invalid data is provided to an endpoint? The system should validate input and return helpful error messages.

- **FR-001**: System MUST provide a database schema with Users, Products, Cart, Wishlist, and Orders tables
- **FR-002**: System MUST implement GET /api/products endpoint to list all available products with search/filter/sort capabilities
- **FR-003**: System MUST implement GET /api/products/:id endpoint to retrieve detailed product information
- **FR-004**: System MUST implement POST /api/cart endpoint to add products to a user's cart
- **FR-005**: System MUST implement GET /api/cart endpoint to retrieve a user's cart items
- **FR-006**: System MUST implement POST /api/wishlist endpoint to add products to a user's wishlist
- **FR-007**: System MUST implement GET /api/wishlist endpoint to retrieve a user's wishlist items
- **FR-008**: System MUST implement POST /api/orders endpoint to create a new order from cart items
- **FR-009**: System MUST store product information including name, description, price, stock, category, and image array
- **FR-010**: System MUST validate input data for all endpoints to prevent invalid data insertion
- **FR-011**: System MUST handle errors gracefully and return appropriate HTTP status codes and messages
- **FR-012**: System MUST connect to Neon Serverless PostgreSQL database using DATABASE_URL environment variable
- **FR-013**: System MUST support USD currency pricing for all products and orders

### Key Entities *(include if feature involves data)*

- **User**: Represents a customer with email and password hash for authentication (to be implemented in Spec 3)
- **Product**: Represents items available for purchase with name, description, price, stock level, category, and image array
- **Cart**: Represents items in a user's shopping cart with user_id, product_id, and quantity
- **Wishlist**: Represents items saved by user for future consideration with user_id and product_id
- **Order**: Represents a completed purchase with user_id, total amount, status, shipping address, and items as JSON array

- Q: How long should cart items remain before automatic removal? → A: Cart items expire after 24 hours (restore stock to inventory)
