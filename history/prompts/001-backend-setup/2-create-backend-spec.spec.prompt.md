## Prompt

Spec 1 - Backend & Database Setup


Core goal: Set up the Express.js backend and Neon PostgreSQL database for the Homestore e-commerce website, including models, endpoints, and basic CRUD operations for products, orders, cart, and wishlist.

Key requirements:
- Use Express.js with TypeScript
- Connect to Neon Serverless PostgreSQL (DATABASE_URL in .env)
- Use Prisma ORM (or SQLModel if preferred) for database models
- Database models (tables):
  - Users (id, email, password_hash, created_at)
  - Products (id, name, description [generated by Claude], price, stock, category, images [array])
  - Cart (id, user_id, product_id, quantity, created_at)
  - Wishlist (id, user_id, product_id, created_at)
  - Orders (id, user_id, total_amount, status, shipping_address, created_at, items [JSON])
- Basic endpoints (protected later in Spec 3):
  - GET /api/products → list all products (with search/filter/sort query params)
  - GET /api/products/:id → product detail
  - POST /api/cart → add to cart
  - GET /api/cart → get cart items
  - POST /api/wishlist → add to wishlist
  - GET /api/wishlist → get wishlist items
  - POST /api/orders → create order (simulated checkout)
- Use environment variables (.env.example) for DATABASE_URL
- Add basic error handling and input validation

Constraints:
- Use existing homestore-sparkle demo as reference only for UI (no backend there)
- No authentication middleware yet (added in Spec 3)
- Currency: USD (fixed in price field)
- Product descriptions: Claude will generate later — for now use placeholder text
- Product images: placeholders for now (provided later)

Success criteria:
- Backend runs (node server.js)
- Prisma migrate works → tables created in Neon DB
- Postman test: GET /api/products → returns list
- POST /api/cart → adds item to cart
- All endpoints return correct data

Use Context7 MCP for Express.js + Prisma/Neon setup examples if needed.
